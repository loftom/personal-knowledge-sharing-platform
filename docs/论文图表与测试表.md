## 2. 数据库 E-R 图（核心实体）

```mermaid
erDiagram
  USER ||--o{ CONTENT : publishes
  USER ||--o{ COMMENT : writes
  USER ||--o{ PRIVATE_MESSAGE : sends
  USER ||--o{ PRIVATE_MESSAGE : receives
  USER ||--o{ NOTIFICATION : owns

  CONTENT ||--o{ COMMENT : has
  CONTENT ||--o{ CONTENT_TAG : links
  TAG ||--o{ CONTENT_TAG : maps

  CONTENT ||--|| QA_QUESTION : extends
  QA_QUESTION ||--o{ QA_ANSWER : has
  USER ||--o{ QA_ANSWER : answers

  USER {
    bigint id PK
    varchar username
    varchar nickname
    varchar role
    int status
  }

  CONTENT {
    bigint id PK
    bigint author_id FK
    varchar type
    varchar title
    varchar status
    varchar visibility
  }

  QA_QUESTION {
    bigint id PK
    bigint question_id FK
    varchar status
    bigint best_answer_id
  }

  QA_ANSWER {
    bigint id PK
    bigint question_id FK
    bigint user_id FK
    int is_best
  }
```

## 3. 关键流程图

### 3.1 内容发布与审核流程

```mermaid
flowchart TD
  A[用户提交内容] --> B[内容入库 status=PENDING_REVIEW]
  B --> C{管理员审核}
  C -->|通过| D[status=PUBLISHED]
  C -->|驳回| E[status=REJECTED]
  D --> F[首页/推荐可见]
  E --> G[作者在个人中心查看并修改]
```

### 3.2 问答最佳答案流程

```mermaid
flowchart TD
  A[用户提交回答] --> B[问答状态保持 PENDING]
  B --> C{提问者/管理员选择最佳答案}
  C -->|是| D[答案 is_best=1]
  D --> E[问答状态 RESOLVED]
  E --> F[触发通知与实时事件]
```

### 3.3 私信实时推送流程

```mermaid
flowchart TD
  A[发送方调用 /api/message] --> B[后端写入 private_message]
  B --> C[后端写入 notification]
  C --> D[RealtimePushService 推送事件]
  D --> E[前端 WebSocket 收到 private-message/notification]
  E --> F[会话列表与未读数实时刷新]
```
